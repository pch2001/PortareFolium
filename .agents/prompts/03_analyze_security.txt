작업을 시작하기 전에 반드시 `AGENTS.md`와 `agents/` 폴더 내의 지침을 숙지할 것.

[목표: 보안 취약점 분석]
여기에 분석할 대상을 적기 (예: Server Actions 내 NextAuth 관리자 세션 검증 로직 검토)

[자율 실행 지침 - 절대 규칙]
1. 분석 우선: 코드를 즉시 수정하지 말아줘. 구조적 취약점(예: 권한 우회, SSR/CSR 데이터 유출, API 오용)을 먼저 꼼꼼히 탐색해줘.
2. 프로젝트 특성 반영: `serverClient`(service_role)와 `browserClient`(anon+RLS)가 아키텍처 규칙(`agents/05-architecture.md`)에 맞게 적절히 격리되어 사용되었는지 집중적으로 확인해줘.
3. 보고서 작성: 취약점 발견 시 `SECURITY_AUDIT.md` 파일을 생성하여 발견된 문제점과 제안하는 해결책을 리스트업해줘.
4. 분석 보고서 작성이 끝나면 나에게 알려줘. 내가 승인한 항목에 대해서만 후속 수정 작업을 진행할 거야.
