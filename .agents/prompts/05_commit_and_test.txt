[목표: 커밋 전 최종 검증 및 커밋 메시지 작성]
현재 스테이징된 작업물 또는 방금 수정된 내용의 마무리 점검을 진행해줘.

[자율 실행 지침 - 절대 규칙]
1. 자율 테스트 루프: 로컬 commit gate인 `pnpm exec vitest run`과 `pnpm build`를 스스로 실행해줘. 실패 시 최소 3회 스스로 원인을 찾아 디버깅해줘.
2. E2E 환경 인식: 런타임 에러나 Playwright 테스트 연관성이 있다면 검증 계획을 `TODO.md`에 세워줘.
3. 커밋 분리: 무관한 파일 변경사항이 묶여있다면 논리적으로 커밋 단위를 분리할 것을 제안해줘.
4. 커밋 메시지 작성: 모든 검증이 완벽히 끝나면 `<type>: <Korean description> (v<version>)` 포맷(`agents/04-workflow.md` 참고)에 맞춰 git commit을 수행하거나 커밋 메시지를 텍스트로 제시해줘.
